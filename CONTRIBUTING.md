# 📋 **Guía de Contribución** 📋

</div>

---

Gracias por tu interés en contribuir a nuestros proyectos de código abierto. Esta guía está diseñada para contribuyentes externos que desean participar en nuestros repositorios públicos.

---

## Antes de Comenzar

1. Revisa el archivo `README.md` del repositorio para entender el contexto del proyecto.
2. Explora la documentación existente antes de proponer cambios mayores.
3. Si planeas un cambio significativo, considera abrir un issue primero para discutir la propuesta.

---

## Proceso de Contribución

### 🐛 Reporte de Bugs

- Utiliza la plantilla de issues del repositorio para reportar bugs.
- Incluir pasos para reproducir, comportamiento esperado y actual.
- Si descubres una vulnerabilidad de seguridad, consulta `SECURITY.md`.

### ✨ Propuestas de Mejoras

- Verifica que la mejora no haya sido propuesta anteriormente.
- Describe claramente el problema que resuelve y el enfoque propuesto.
- Incluye mockups, diagramas o ejemplos si aplica.

### 🔧 Pull Requests

- Mantén las PRs pequeñas y enfocadas en un solo tema.
- Incluye descripción clara de qué cambia y por qué.
- Asegúrate de que el código pase las verificaciones automáticas.
- Espera feedback del equipo de revisión antes de hacer push.

---

## Convenciones de Commits

Seguimos commits descriptivos y significativos. Consulta la documentación específica del proyecto para conocer las convenciones establecidas.

---

## Revisión de Código

El equipo de mantenimiento revisará las contribuciones y proporcionará feedback. Sé abierto a la discusión y disposto a hacer ajustes según las sugerencias recibidas.

---
